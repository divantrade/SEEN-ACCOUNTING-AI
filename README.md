# SEEN Accounting AI

نظام محاسبة ذكي متكامل مبني على Google Sheets مع بوت Telegram تقليدي وبوت ذكاء اصطناعي،
مصمم خصيصاً لإدارة حسابات الإنتاج الإعلامي لشركة **Start Scene Media Produksiyon Limited**.

## المميزات

### المحاسبة والحركات المالية
- تسجيل الحركات المالية (مدين / دائن / تسوية) مع دعم متعدد العملات (USD, TRY, EGP)
- إدارة الموردين والعملاء والممولين
- تتبع المشاريع والبنود والوحدات الإنتاجية
- نظام الاستحقاقات والتنبيهات للمدفوعات القادمة
- تسوية بنكية ومتابعة النقدية
- نظام مراجعة واعتماد الحركات المُدخلة من البوت

### التقارير
- كشوفات حساب PDF احترافية (مورد / عميل / ممول)
- تقارير ربحية المشاريع وتكلفة الوحدات الإنتاجية
- القوائم المالية (قائمة الدخل، الميزانية العمومية، ميزان المراجعة)
- دفتر الأستاذ العام وقيود اليومية وشجرة الحسابات
- تقارير المصروفات والإيرادات والتدفق النقدي
- تقارير عمولات مديري المشاريع
- الموازنات التخطيطية ومقارنتها بالفعلي
- لوحة تحكم (Dashboard) شاملة

### بوت Telegram التقليدي
- تسجيل حركات مالية عبر قوائم تفاعلية وأزرار
- 18 حالة محادثة (Conversation States) لضمان اكتمال البيانات
- 15+ لوحة مفاتيح تفاعلية لأنواع الحركات والعملات وطرق الدفع
- إرفاق مستندات وصور مع الحركات

### بوت الذكاء الاصطناعي
- تسجيل حركات بالمحادثة الطبيعية بالعربية
- يعمل بنموذج Gemini 2.0 Flash مع 8 نماذج احتياطية
- بحث ذكي بالعربية مع دعم الأخطاء الإملائية (Fuzzy Search)
- إصدار تقارير وكشوفات حساب PDF مباشرة من البوت

## التثبيت والإعداد

### 1. استنساخ المشروع
```bash
git clone https://github.com/divantrade/SEEN-ACCOUNTING-AI.git
cd SEEN-ACCOUNTING-AI
```

### 2. تثبيت وتسجيل الدخول لـ clasp
```bash
npm install -g @google/clasp
clasp login
```

### 3. ربط المشروع بـ Google Apps Script
ملف `.clasp.json` يحتوي على `scriptId` المرتبط بالمشروع. إذا كنت تبدأ من جديد:
```bash
clasp clone <SCRIPT_ID>
```

### 4. رفع الكود
```bash
clasp push
```

### 5. إعداد Telegram Bots
1. أنشئ بوتين عبر [@BotFather](https://t.me/BotFather) (تقليدي + ذكي)
2. احفظ الـ Tokens في Script Properties من Google Apps Script:
   - `TELEGRAM_BOT_TOKEN` - توكن البوت التقليدي
   - `AI_BOT_TOKEN` - توكن البوت الذكي
   - `GEMINI_API_KEY` - مفتاح Gemini API
3. اضبط Webhook أو فعّل Time-driven Trigger لـ `processAIBotUpdates()`

### 6. إعداد المستخدمين
أضف المستخدمين المصرح لهم في شيت "المستخدمين المصرح لهم" مع تحديد الصلاحيات:
- بوت تقليدي / بوت ذكي / شيت / مراجعة

## خريطة التطوير: التحول إلى SaaS

تم التخطيط لتحويل هذا النظام من Google Sheets إلى منتج SaaS مستقل. يوجد فرعان يوثقان هذا العمل:

### المحاولة الأولى: KodLab SaaS (فرع `claude/excel-to-web-app-lhzfF`)

**الحالة:** تخطيط فقط - لم يبدأ التنفيذ

**الرؤية:** تحويل النظام إلى منتج SaaS تجاري متعدد المستأجرين (Multi-tenant) باسم **KodLab** يخدم أي صناعة.

| الحالي (SEEN) | المخطط (KodLab) |
|---|---|
| Google Sheets | PostgreSQL (Supabase) |
| بوت Telegram فقط | واجهة ويب كاملة + بوت |
| مستخدم واحد | عملاء متعددين (Multi-tenant) |
| للإنتاج الفني فقط | لأي صناعة |
| مجاني | اشتراك شهري |

**التقنيات المقترحة:** Next.js + TypeScript + Tailwind CSS + Supabase + Vercel

**خطة التنفيذ المقترحة:**
1. إنشاء حسابات (Supabase + Vercel) + مشروع Next.js
2. قاعدة البيانات + واجهة الويب (Login + Dashboard)
3. البوت الذكي + الإدخال الذكي (Guided Entry)
4. التقارير + الاشتراكات + الإطلاق

---
